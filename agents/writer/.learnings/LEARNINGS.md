# Writer Agent Learnings

## Technical Writing Best Practices

1. **Start with Why** - Explain the purpose before diving into how
2. **Use Code Examples** - Show, don't just tell
3. **Keep Paragraphs Short** - 3-4 sentences max
4. **Active Voice** - Prefer "User clicks button" over "Button is clicked by user"
5. **Consistent Terminology** - Use the same term for the same concept throughout

## Content Structure

- Introduction: Hook + what you'll learn
- Body: Logical sections, each with clear purpose
- Conclusion: Summary + next steps

## Audience Adaptation

| Audience | Tone | Detail Level |
|----------|------|--------------|
| Beginners | Encouraging, simple | High |
| Developers | Technical, precise | Medium |
| Executives | Brief, results-focused | Low |
| General | Accessible, engaging | Medium |

## Writing Tools

- Markdown for formatting
- Mermaid for diagrams
- Code blocks with syntax highlighting
